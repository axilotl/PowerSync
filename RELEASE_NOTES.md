## What's Changed

**Fix: TOU sync silently stopped running after upgrading to v2.12.208 for Amber, Localvolts, and Flow Power non-AEMO users**
The new AEMO-dispatch-driven tariff sync added in v2.12.208 only worked for one specific configuration — Flow Power users running the AEMO direct price source. Everyone else (Amber, Localvolts, and Flow Power users on Amber/Sentinel price sources) was getting no Tesla tariff POSTs at all because the dispatch signal that triggers the sync is only emitted by the AEMO price coordinator, and that coordinator was only constructed for Flow Power AEMO mode. The result was Tesla holding whatever tariff was last uploaded before the upgrade — meaning Powerwall's "utility buy" / "utility sell" prices in the Tesla app could disagree with PowerSync's live prices indefinitely, and the "PowerSync changed Utility Rate Plan" log entry stopped appearing entirely. Manual Force Charge / Force Discharge still worked because they go through a different path. PowerSync now starts a dispatch-trigger AEMO coordinator for any NEM-on provider (Amber, Localvolts, Flow Power) that doesn't already have one. The NEM region is auto-detected from existing config first, then from the Tesla site's `installation_time_zone` (`Australia/Sydney` → `NSW1`, `Australia/Melbourne` → `VIC1`, etc.), then from Amber's network field — so Tesla users get correct dispatch-driven sync without needing to pick a region in PowerSync settings. If no region can be resolved, a periodic fallback at :40s past every 5-minute boundary keeps tariff sync running. Octopus UK is unaffected; it has its own :00/:30 cron.
